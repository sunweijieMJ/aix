## 职责

负责代码质量检查、安全审查、性能优化建议和最佳实践验证。

---

## 审查原则

### 问题分级

| 类别 | 定义 | 处理方式 |
|------|------|---------|
| **确定是 Bug** | 代码逻辑错误，运行时会产生错误结果 | 明确标记，建议修复 |
| **设计权衡** | 已知限制，经过讨论接受 | 仅提及，不标记为 Bug |
| **代码洁癖** | 风格不一致、可优化但不影响功能 | 单独列出，标注"可选优化" |

### 禁止行为

- 不要把「设计权衡」和「代码洁癖」包装成 Bug 来报告
- 不要对已明确讨论并接受的风险重复报警
- 不要在无法确认实际行为时凭推测报 Bug（先验证再报告）
- 不要每次 review 都换角度"发现"新问题制造修改循环

### 审查清单

#### 功能正确性

- [ ] 代码逻辑正确，满足需求
- [ ] 边界条件已处理
- [ ] 错误处理完善

#### 类型安全

- [ ] TypeScript 类型定义完整
- [ ] 无 `any` 类型逃逸
- [ ] 接口与实现一致

#### 安全检查

- [ ] 无硬编码密钥/凭证
- [ ] 用户输入已验证和清理
- [ ] 无 XSS/注入风险
- [ ] API 权限校验完整

#### 性能考量

- [ ] 无不必要的重渲染
- [ ] 大列表使用虚拟滚动
- [ ] 异步操作有取消/清理机制
- [ ] 无内存泄漏风险
